# DuckH - لعبة صيد البط

لعبة صيد البط ثلاثية الأبعاد مطورة باستخدام C++20 و Vulkan للرسومات.

## الميزات

- **رسومات ثلاثية الأبعاد**: باستخدام Vulkan لأداء عالي
- **أنيميشن متقدم**: نظام أنيميشن مرن للبط والصياد
- **صوت**: دعم صوتي باستخدام miniaudio
- **واجهة مستخدم**: نصوص عربية وأسماء الله الحسنى
- **أنظمة لعب**: نظام مكافآت، تقييم مهارة، إدارة دفعات
- **نظام المهارات**: تقييم أداء اللاعب وتصنيف المهارة
- **منصة macOS**: مُحسّن لنظام macOS

## نظام المهارات

يحتوي اللعبة على نظام تقييم مهارة متقدم يقيس أداء اللاعب بناءً على:

- **الدقة**: نسبة الإصابات الناجحة
- **الكفاءة**: عدد البط المصابة مقابل الطلقات المستخدمة
- **الوقت**: سرعة إكمال المراحل

مستويات المهارة:
- ضعيف (Poor)
- مقبول (Average)
- جيد (Good)
- ممتاز (Excellent)
- احترافي (Perfect)

## متطلبات البناء

- macOS 11.0 أو أحدث
- CMake 3.16 أو أحدث
- Xcode 12.0 أو أحدث (مع أدوات سطر الأوامر)
- Vulkan SDK

## كيفية البناء

```bash
# استنساخ المستودع
git clone <repository-url>
cd duckh

# إنشاء مجلد البناء
mkdir build
cd build

# تكوين المشروع
cmake .. -DCMAKE_BUILD_TYPE=Release

# البناء
cmake --build . --config Release
```

## كيفية التشغيل

```bash
# من مجلد البناء
open Duck.app
```

## هيكل المشروع

```
src/
├── core/          # النواة الأساسية (التطبيق، النافذة، الإدخال، إلخ)
├── gameplay/      # منطق اللعب (البط، الصياد، المكافآت، إلخ)
├── rendering/     # الرسومات (Vulkan، المشهد، التخطيط)
├── assets/        # الأصول (الأنيميشن، الأطالس، الطبيعة)
├── audio/         # الصوت
├── ui/            # واجهة المستخدم (النصوص)
└── gfx/           # الرسومات الأساسية (Vulkan، stb_image)
```

## نظام التطوير

يستخدم المشروع نظام وكيل تطوير ذكي (Agent) للمساعدة في:

- إضافة ميزات جديدة
- إصلاح الأخطاء
- تحسين الأداء
- صيانة الكود

الوكيل يعتمد على ملفات الإرشادات:
- `AGENT.md`: تعليمات الوكيل
- `SKILL.md`: مهارات التطوير
- `references/duckh-map.md`: خريطة المشروع

## التقنيات المستخدمة

- **C++20**: اللغة الرئيسية
- **Vulkan**: API الرسومات
- **GLFW**: إدارة النوافذ والإدخال
- **miniaudio**: معالجة الصوت
- **stb_image**: تحميل الصور
- **CMake**: نظام البناء

## المساهمة

يرجى اتباع إرشادات الجودة:
- استخدام C++20 الحديث
- تعليقات عربية للوثائق
- اختبار شامل قبل الإرسال
- الحفاظ على هيكل المشروع النظيف
